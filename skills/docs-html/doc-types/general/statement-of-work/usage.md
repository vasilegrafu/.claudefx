# statement-of-work (SOW)

A contractual scope agreement: parties, deliverables, schedule, acceptance,
price. Signed. More formal and bounded than a project plan.

- Audience: client + supplier signatories. Altitude: contractual — precise,
  unambiguous, binding.
- Filename: `docs/statement-of-work-<engagement>.html`
- Template: `document.html.j2` (in this folder)
- Depth: `full` (it is a structured form — fill the fields, do not research
  code).

## Rules
- Deliverables are concrete and acceptance-testable — each has an acceptance
  criterion.
- Assumptions and exclusions are explicit; ambiguity is a contract risk.
- Ends with the [[approval-block]] — signatures on paper, empty cells.
