# roadmap

- Audience: everyone steering by it — a living document, updated on cadence.
- Filename: `docs/roadmap-<subject>.html` (or `-<year>`).
- Depth: none — the owner's plan.
- Rules: goals are outcomes with done-conditions, not activity lists; the
  timeline marks done/current/planned honestly; date changes are edits with a
  change-history row, never silent; a mermaid `gantt` diagram is a fine
  alternative to the timeline for many parallel tracks.
