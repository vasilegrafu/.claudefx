# disaster-recovery-plan

- Audience: whoever is recovering the system, under stress, possibly without
  the person who wrote it.
- Filename: `docs/disaster-recovery-plan-<system>.html`.
- Depth: full.
- Rules: RTO/RPO stated as numbers up front; procedures are step-lists
  executable verbatim; a backup that has never been restore-TESTED does not
  count as a backup — the Testing section records real test dates and
  results.
