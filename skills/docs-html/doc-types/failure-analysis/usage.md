# failure-analysis

- Audience: engineering and management deciding what changes after a failure
  (the physical-world sibling of incident-postmortem).
- Filename: `docs/failure-analysis-<item>-<date>.html`.
- Depth: ask.
- Rules: evidence before analysis, analysis before cause — never start from
  the conclusion; rejected hypotheses stay in the document (they are the
  proof of rigor); root cause is distinguished from contributing factors;
  every corrective action has an owner and a deadline.
