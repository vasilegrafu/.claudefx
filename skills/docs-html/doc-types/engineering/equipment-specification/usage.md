# equipment-specification

- Audience: suppliers quoting against it and inspectors accepting against it.
- Filename: `docs/equipment-specification-<equipment>.html`.
- Depth: full.
- Rules: requirements are `requirement` cards with EQ- ids — parameter,
  value, tolerance, units, and a verification method each; acceptance in
  Given/When/Then form; specify WHAT is required, not HOW to build it, unless
  the how is genuinely mandatory.
